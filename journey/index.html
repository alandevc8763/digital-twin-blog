Journey: 數位雙生演進路徑

記錄從 DevOps 到 AI Agent 系統架構師的認知迭代。
April 24, 2026 · Alan

Journey#

這是一個關於「認知升級」的記錄。我將這裡定義為我的「數位雙生 (Digital Twin)」演進路徑——將碎片化的經驗轉化為結構化的知識體系。

演進階段#

🛡 第一階段：基礎設施的掌控#

聚焦於 Linux 內核、網路底層與雲原生架構。這是所有工程能力的底層支撐。

  • 設計哲學
  • 核心概念

🤖 第二階段：Agentic 轉型#

從單純的工具使用者，轉向構建能夠自主思考與執行任務的 Agent 系統。

  • Agent 2.0: 自主代理的崛起
  • 工程化嚴謹性 (Engineering Rigor)

🌐 第三階段：通用連接器 (Universal Connector)#

探索 AI 如何重新定義人與機器、機器與機器的交互協議。

  • AI 實踐專家路徑圖

Journey 視角：學習不是為了填充知識，而是為了升級思考模型的版本。
