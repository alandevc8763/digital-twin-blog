「數位分身」部落格實作計畫#

給 Hermes 的指令： 請使用 subagent-driven-development 技能，逐項執行此計畫中的任務。

核心目標： 構建一個專業且結構化的知識庫（基於 MkDocs Material），作為 Alan 的數位分身（Digital Twin）。該網站將部署於 GitHub Pages，採取「精英（Elite）」視覺風格，並將所有高價值內容重新編寫。

架構設計：

  • 靜態網站生成器： MkDocs 搭配 Material 主題。
  • 視覺美學： 自定義 CSS 以實現「金箔鑲石板 (Gold-on-Slate)」風格（深石板色背景搭配金箔色點綴）。
  • 內容管線： 從 sz9751210.github.io 提取原始內容 → 重新編寫為「精英格式」 → 生成 Markdown 檔案。
  • 部署方案： 透過 GitHub Actions 自動部署至 GitHub Pages。

技術棧：

  • Python 3.11+
  • mkdocs, mkdocs-material
  • GitHub Actions
  • Custom CSS

第一階段：基礎設施與環境搭建#

任務 1：初始化專案目錄#

目標： 建立基礎文件夾結構。

  • 創建：~/digital-twin-blog/docs
  • 創建：~/digital-twin-blog/overrides/main.css

任務 2：安裝相依套件#

目標： 確保所有必要套件已就緒。

  • 執行：pip install mkdocs mkdocs-material
  • 預期結果：SUCCESS

任務 3：MkDocs 基礎初始化#

目標： 生成預設配置檔案。

  • 在 ~/digital-twin-blog 執行：mkdocs new .
  • 預期結果：生成 mkdocs.yml 與 docs/index.md。

任務 4：配置 Material 主題#

目標： 啟用 Material 主題與深色模式。

  • 修改：~/digital-twin-blog/mkdocs.yml
  • 添加配置：
theme:
  name: material
  palette:
    - scheme: slate
      primary: gold
      accent: gold
  features:
    - navigation.tabs
    - navigation.sections
    - content.code.copy

任務 5：實作「金箔鑲石板」視覺效果#

目標： 套用自定義的精英視覺風格。

  • 修改 ~/digital-twin-blog/mkdocs.yml 以包含：
extra_css:
  - overrides/main.css
  • 修改 ~/digital-twin-blog/overrides/main.css 添加：
:root {
  --md-primary-fg-color: #D4AF37; /* Gold */
  --md-accent-fg-color: #D4AF37;
}
.md-header {
  background-color: #1a1a1a;
}

第二階段：分類體系與結構設計#

任務 6：定義導航分類體系 (Taxonomy)#

目標： 根據專業核心能力建立邏輯結構。

  • 修改 ~/digital-twin-blog/mkdocs.yml
  • 添加 nav 區段：
    • 雲端基礎設施 (AWS, MSK, IAM)
    • 資料庫工程 (MongoDB, Optimization)
    • 自動化與工具開發 (Python, Telegram Bots)
    • 數位分身演進路徑 (Reflection, Vibe Coding)

任務 7：建立文件夾結構#

目標： 在文件系統中映射導航結構。

  • 創建目錄：
    • docs/cloud_infra/
    • docs/db_eng/
    • docs/automation/
    • docs/journey/

第三階段：內容工程（數位分身核心）#

任務 8：原始內容提取#

目標： 從 sz9751210.github.io 提取舊有文章。

  • 動作：使用瀏覽器讀取文章 → 將原始文本保存至 docs/raw/。

任務 9：將內容重構為「精英格式」#

目標： 將碎片化文章轉化為高價值的知識資產。

  • 寫作模板：
    • 分類體系 (Taxonomy)：(領域/信號)
    • 核心洞察 (Core Insight)：(這件事的核心價值是什麼？)
    • 實作詳情 (Implementation/Details)：(技術實作路徑)
    • 協同效應 (Synergy)：(與其他知識點的關聯)
  • 儲存路徑：docs/[category]/[post_name].md

任務 10：創建首頁（定義數位分身身份）#

目標： 定義「數位分身」的人格特質。

  • 修改：docs/index.md
  • 內容：包含專業總結、目前的成長軌跡，以及指向預研腦暴紀錄的連結。

第四階段：部署管線#

任務 11：初始化 GitHub 倉庫#

目標： 建立遠端託管空間。

  • 執行：git init, git add ., git commit -m "init: digital twin blog"
  • 動作：透過 GH CLI/API 創建 digital-twin-blog 倉庫並推送。

任務 12：設定 GitHub Actions 自動部署#

目標： 將構建與部署流程自動化。

  • 創建：.github/workflows/deploy.yml
  • 內容：使用 MkDocs-Material 標準部署工作流，推送至 gh-pages 分支。

任務 13：最終驗證#

目標： 端到端檢查。

  • 執行：mkdocs serve → 檢查視覺效果 → 推送到 GitHub → 確認 Live URL 正常訪問。
