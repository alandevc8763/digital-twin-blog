    打造私有雲的核心：Proxmox VE 的超融合實踐

    當你的 HomeLab 從「跑幾個 Docker」演進到「運行多個獨立服務」時，你會發現單純的 OS 層級管理已經無法滿足需求。你需要的是一個 超融合基礎設施 (HCI) —— 讓計算、儲存、網路在單一界面中完成資源調度。

    Proxmox VE (PVE) 正是這個角色。它將 Debian 的穩定性、KVM 的強大隔離與 ZFS 的數據可靠性完美整合。

    KVM vs LXC：選擇困難症的終結

    很多新手在 PVE 中糾結該開 VM 還是 LXC。其實這是一個簡單的 隔離度 vs 效率 的權衡。

    • KVM (虛擬機)：給它完整的硬體模擬。你需要運行 Windows、FreeBSD 或者需要深度修改內核的服務？選 KVM。它提供最強的隔離，但代價是需要獨立的 Guest OS，啟動慢且吃記憶體。
    • LXC (容器)：與主機共享內核。如果你只需要跑個 Nginx、Pi-hole 或 Home Assistant，LXC 是神級選擇。它幾乎沒有額外開銷，秒級啟動，資源利用率極高。

    我的 SRE 經驗法則：除非有明確的 OS 要求或極高的安全隔離需求，否則優先考慮 LXC。

    讓硬體「突破」虛擬化：PCIe Passthrough

    Proxmox 最迷人的地方在於 硬體透傳。你可以將一張 RTX 4090 直接「交給」一個 VM，讓該 VM 擁有原生的 GPU 性能來跑 LLM 或轉碼。

    但這裡有個坑：IOMMU Groups。硬體在物理上是分組的，如果你想透傳一個設備，而它與另一個關鍵設備在同一組，你可能得把整個組都透傳出去。

    SRE 避坑指南： 1. 檢查 CPU 的 VT-d 或 AMD-Vi 是否開啟。 2. 在 /etc/default/grub 中正確配置 intel_iommu=on。 3. 使用 vfio-pci 在啟動時就將設備「綁定」，防止 PVE 主機搶佔硬體。

    PBS：數據主權的最後一道防線

    如果說 PVE 是身體，那麼 PBS (Proxmox Backup Server) 就是靈魂。

    傳統備份是全量複製，而 PBS 採用 塊級去重 (Block-level Deduplication)。想像一下，你有 10 個 Ubuntu VM，它們的 OS 基礎部分幾乎完全一樣。PBS 只會儲存一份基礎鏡像，其餘 VM 只儲存差異部分。這不僅極大節省空間，還讓備份速度快到不可思議。

    穩健節點配置清單

    • [ ] Boot Drive：使用 Enterprise 級 SSD 鏡像。不要用消費級 SSD 跑 ZFS，否則會被寫入放大（Write Amplification）搞到崩潰。
    • [ ] 網路分流：將管理流量 (Management)、虛擬機流量 (VM Traffic) 與儲存同步流量 (Replication) 分開到不同的 VLAN。
    • [ ] Quorum 意識：如果你搭建叢集，請至少準備 3 個節點。否則一旦一台宕機，剩下的節點會因為失去「法定人數 (Quorum)」而進入 Read-only 模式。
